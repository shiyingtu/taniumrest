Running Actions - Examples

Run Query

The run query action can be used to either run previously existing Saved Questions or can run ad-hoc manual questions.

Example Parameters - Manual Questions

Example 1 - Get Computer Name timeout_seconds = 600 is_saved_question = false query_text = Get Computer Name from all machines group_name =

Example 2 - Get Computer Name for Specified Computer Group timeout_seconds = 600 is_saved_question = false query_text = Get Computer Name from all machines group_name = centos-computers

Example 3 - A Complex Query timeout_seconds = 600 is_saved_question = false query_text = Get Trace Executed Processes[1 month,1522723342293|1522726941293,0,0,10,0,rar.exe,"",-hp,"","",""] from all machines group_name =

Example 4 - List Process Details for a Specified Device timeout_seconds = 600 is_saved_question = false query_text = Get Process Details["",""] from all machines with Computer Name contains localhost

Example Parameters - Saved Question

Saved questions are queries that are pre-configured and saved through the Tanium UI. A saved question can be invoked by providing the name of the saved question as opposed to the query string itself.

  • The is_saved_question flag must be set to true when running a saved question.

  • The group_name parameter is ignored for saved questions.

Example 1 - Run Saved Question

In this example, we are running a Saved Question entitled 'splunk-asset-data'. 

timeout_seconds = 600
is_saved_question = true
query_text = splunk-asset-data
Number of results: 0

Run Action

The 'Run Action' action will cause a specified Tanium Package to be executed on the specified group.

Any parameters required by the specified package must be supplied a valid JSON via the action_paramters parameter. For example, {"$1":"Standard_Collection", "$2":"SCP"}

Example 1 - Run 'Live Response - Linux' Package expire_seconds = 600 package_name = Live Response - Linux package_parameters = {"$1":"Standard_Collection", "$2":"SCP"} action_group = Default action_name = matt-test-action-name group_name = centos-computers Successfully executed the action
